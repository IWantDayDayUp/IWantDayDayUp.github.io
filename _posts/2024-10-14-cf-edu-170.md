## D. Attribute Checks (DP)

[D. Attribute Checks](https://codeforces.com/contest/2025/problem/D)

Define $dp[i][I]$: the answer to the task if we processed first $i$ records and the current Intelligence level is $I$.

- $P$: a total number of points in first $i$ records
- $I$: the current Intelligence level
- $S = P - I$: the current Strength level

For dp transitions:

- If `the last record $r_i = 0$`: then it was a point, and there are only two options:
  - we either raised Intelligence, so the last state was $dp[i - 1][I - 1]$
  - or raised Strength, the last state was $dp[i - 1][I]$
  - we can calculate `$dp[i][I] = max(dp[i - 1][I - 1], dp[i - 1][I])$`
- If `the last record $r_i > 0$`: then it was an Intelligence check, and it doesn't affect the state, only it's answer.
  - For all $I \ge r_i$, $dp[i] = dp[i - 1][I] + 1$
  - for all $I < r_i$, $dp[i] = dp[i - 1][I]$
- If `the last record $r_i < 0$`: then it was a Strength check and affect the values in a similar way.
  - For all $I \le P + r_i$, $dp[i] = dp[i - 1][I] + 1$
  - for all $I > P + r_i$, $dp[i] = dp[i - 1][I]$

So we got a solution with $O(nm)$ time and memory, but we can speed it up.

- `Reducing the time complexity`: the second and third cases are just `range addition`, so, instead of adding some value $v$ to the segment $[l, r]$, we'll only add value $v$ to $a[l]$ and $-v$ to $a[r + 1]$. And when we meet $r_i = 0$, we'll push all accumulated operations all at once.
- `Reducing the space complexity`: As usual, we can store only the last two layers: for the current layer $i$ and the previous layer $i - 1$. But actually, we can store only one layer and update it in-place.
  - store only one layer of dp as $dp[I]$, attribute checks don't change $dp$ at all
  - When $r_i = 0$, firstly, push all data from $a$ to $b$, and then we need to recalc values in $dp$.
  - But since the formula is $dp[I] = max(dp[I], dp[I - 1])$, we can just iterate over $I$ in descending order and everything works!

In total, we have a solution with $O(m^2 + n)$ time and $O(m)$ space complexity

```c++
#include <bits/stdc++.h>

using namespace std;

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n, m;
    cin >> n >> m;

    vector<int> dp{0};
    vector<int> f{0};
    int s = 0;

    for (int i = 0; i < n; i++)
    {
        int r;
        cin >> r;

        if (r > 0)
        {
            if (r < f.size())
            {
                f[r]++;
            }
        }
        else if (r < 0)
        {
            r = -r;
            r = s - r;
            if (r >= 0)
            {
                f[0]++;
                f[r + 1]--;
            }
        }
        else
        {
            for (int j = 1; j <= s; j++)
            {
                f[j] += f[j - 1];
            }
            for (int j = 0; j <= s; j++)
            {
                dp[j] += f[j];
            }
            dp.push_back(0);
            s++;
            for (int j = s; j >= 1; j--)
            {
                dp[j] = max(dp[j], dp[j - 1]);
            }
            f.assign(s + 1, 0);
        }
    }

    for (int j = 1; j <= s; j++)
    {
        f[j] += f[j - 1];
    }
    for (int j = 0; j <= s; j++)
    {
        dp[j] += f[j];
    }

    int ans = *std::max_element(dp.begin(), dp.end());
    cout << ans << endl;

    return 0;
}
```
